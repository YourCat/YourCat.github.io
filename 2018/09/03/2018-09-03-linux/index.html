Ubuntu 初试及 IDEA&Git&Atom 使用

前言

这是在Ubuntu下使用Atom编写的第一篇博文，主要讲述近日来关于Ubuntu使用上遇到的一些问题，关于在Ubuntu下使用IDEA编译器，配置java环境，XMind相关软件的使用，以及Git的操作，和Github远程仓库进行代码同步，使用Atom编译器进行代码的编写更改。

正文

IDEA的使用及配置jdk变量

由于IDEA编译器良好的跨平台性，只需要先在Ubuntu下配置jdk环境即可，首先在官网下载对应linux版本的jdk

http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html

对其进行解压缩，然后设置环境变量：
ctrl+alt+T进入终端，打开.bashrc: sudo gedit ~/.bashrc
将以下内容拷贝到文档末尾，更改成自己的路径

保存文本，在终端中输入: source ~/.bashrc 生效
检查是否配置成功: java -version
若配置成功，打印以下内容

java version “1.8.0_181”
Java(TM) SE Runtime Environment (build 1.8.0_181-b13)
Java HotSpot(TM) 64-Bit Server VM (build 25.181-b13, mixed mode)

然后在官网下载IDEA编译器，选择without jdk版本

https://www.jetbrains.com/idea/download/#section=linux

在下载目录进入终端，解压到opt:

sudo tar -zxvf ideaIC-2018.2.2.tar.gz -C /opt

进入IDEA目录的bin文件夹下:

cd /opt/ideaIC-2018.2.2.tar.gz/bin

最后输入:

./idea.sh

傻瓜式操作至安装完成。

XMind8的使用

步骤很简单，先在官网下载XMind8

http://www.xmind.net/download/linux/

下载完成后解压，进入解压后目录，终端输入

sudo ./setup

Git&Atom的使用

使用终端安装Git并进行配置:

sudo apt-get install git
git –version
git config –global user.name “XXX”
git config –global user.email “XXX@qq.com“

本地创建SSH Key。在终端输入（用自己的注册邮箱!）:

ssh-keygen -t rsa -C “youremail@example.com“

在主目录下找到.ssh目录，有id_rsa和id_rsa.pub两个文件，是SSH Key的密钥对,然后连接github上的远程仓库:登陆github，在”setting”中找到”SSH Keys”页面，New Key 输入id_rsa.pub中内容，title随意

在自定义目录下建立本地仓库:

mkdir ihan-blog
cd ihan-blog
git init

克隆github上的仓库

git clone git@github.com:YourCat/YourCat.github.io.git

第一次clone会收到警告，确认GitHub的Key的指纹信息是否真的来自GitHub的服务器,输入yes即可

进入clone的项目目录下:

cd YourCat.github.io

灵活使用以下命令完成同步操作

git add example.markdown //添加指定文件到暂存区
git commit -m “description” //提交暂存区到仓库区
git commit -a //提交工作区自上次commit之后的变化到仓库区
git remote add origin git@github.com:YourCat/YourCat.github.io.git //关联远程仓库
git pull origin master //合并远程分支与本地分支
git push origin master //提交到远程仓库进行同步

第一次推送加上-u参数

git push -u origin master

最后，祭出这张图，愿早日熟悉git命令！
