Tomcat安装及Eclipse配置

前言

由于之前手贱删掉了电脑里与Eclipse环境变量有关的东西，导致其不能使用，索性卸掉下了Eclipse JEE用来学习Java Web相关知识，今早又打算安装Tomcat大干一场，结果在这反反复复的安装配置环境变量上卡了一个早上，因为自己的一些脑残行为遇到各种各样奇葩的报错，现将本次经历记录于此，系统整理一下整个正规的配置问题以及遇到的报错采取的解决方案，文中多处内容从大佬那迁移过来~

正文

整个流程应大致分为三步

  • JDK安装和配置
  • Tomcat安装和配置
  • Elipse安装和配置

ps:三无的情况下才按此顺序操作，如果已有其一或二则跳过对应操作。

JDK安装与配置 具体流程参照 https://jingyan.baidu.com/article/bea41d435bc695b4c41be648.html

  1. 在官网上下载JDK并安装
  2. 配置三个环境变量java_home，path和classpath（都写在系统变量里，无则新建，有则加分号）
  3. JDK配置完成后，可在cmd中输入java -version查看java版本检验是否成功

Tomcat安装与配置

  1. 在官网上下载Tomcat并安装 http://tomcat.apache.org/download-80.cgi#8.5.5

  2. 配置四个环境变量CATALINA_HOME、CATALINA_BASE、CATALINA_TMPDIR、path。

  3. Tomcat配置完成后，在cmd中输入startup.bat，之后会弹出tomcat命令框，输出启动日志。也可以找到该文件手动运行。

  4. 打开浏览器输入http://localhost:8080/，如果进入tomcat欢迎界面，那么恭喜你，配置成功。

    ps:注意图中圈圈，在xml中创建用户名和密码。

Eclipse JEE安装与配置

  1. 在官网上下载Eclipse JEE并安装 http://www.eclipse.org/downloads/eclipse-packages/
    如果已有Eclipse，可”升级”为JEE：打开Eclipse，依次选择Help -> Install New SoftWare，在弹出的窗口的 work with 中输入 http://download.eclipse.org/releases/neon (neon是Eclipse版本名)，回车等待一会，就可以看到很多插件，拉到最下面打开 Web, XML, Java EE and OSGi Enterprise Development ，勾选下面所示四个插件，再点击下方的Next 等待安装重启就可以了。其它插件安装类似，只要勾选不同插件即可。

  2. 前面是JDK和Tomcat的配置，现在要用Eclipse把它们连起来！！！
    连JDK见 http://blog.csdn.net/clj198606061111/article/details/11881575
    连Tomcat&项目部署见 http://blog.csdn.net/yerenyuan_pku/article/details/51830104
    ~~ 内容较多直接上链接…严格按照上述操作完成！

一些遇到的问题

如果有不规范的操作，将会导致生成一些奇奇怪怪的问题，所以不要盲目乱点，先与上述步骤比较有没有不一样或未完成的地方。

  1. Tomcat启动startup.bat时闪退原因及解决方案： http://blog.csdn.net/qq496013218/article/details/74517593
    具体原因：tomcat在启动时，需要读取环境变量和配置信息，如果缺少Java环境，即缺少JAVA_HOME环境变量，则会出现闪退现象。

  2. 运行cmd时报错：Error: could not find java.dll Error:could not find java SE Runtime Environment/
    解决方案：http://blog.csdn.net/hyr83960944/article/details/37592041

  3. 运行JSP报错：The superclass “javax.servlet.http.HttpServlet” was not found on the Java Build Path
    解决方案：http://blog.csdn.net/testcs_dn/article/details/36455669

  4. 运行JSP报错：Several ports (8005, 8080, 8009) required by Tomcat v6.0 Server at localhost are already in use.
    解决方案：http://blog.csdn.net/zhouyingge1104/article/details/7243363

  5. 补：今天打开eclipse莫名又弹了错，问题出在了eclipse和jdk的连接上，莫名失效了。
    打开eclispe报错：java was started but returned exit code=2…….
    解决方案：https://jingyan.baidu.com/article/9989c746255627f648ecfe83.html

后记

应严格按照步骤操作，不要认为能正常运行之前的步骤就都是对的了，比如tomcat运行是需要JAVA_HOME变量的，java是不需要配置JAVA_HOME的，所以之前我即使在JAVA_HOME处出了问题，但是Eclipse仍旧能够正常编译运行程序。详见：http://bbs.csdn.net/topics/390696441

结： 本人菜鸡，如有错误之处望大佬指出多多包涵 QAQ
